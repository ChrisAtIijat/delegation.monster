FAQ Where do you store my data? Who has access to it? The app stores all data locally within the browser. NO data is stored online. NO data is forwarded to any backend. In fact there is NO communication with any backend whatsoever.

delegation.monster is a JavaScript app written in Angular that runs purely and locally within the browser.

The only communication is with the relay defined in the Nostr Connect URI. How do you store my data locally? Most of the data is stored within the browser's IndexedDB . We use RxDB to programmatically access the IndexedDB from within our app. Additionally, some data is stored in the browser's localStorage . Do you use any encryption? How secure are my keys? When you first run our app, you are asked to provide a password that we then use to create the IndexedDB. You have to provide this password every time you start the app.

All secret keys are stored encrypted in the IndexedDB. All other data is not encrypted. You can find more details about the encryption with RxDB here. There is a "just test the software" option where I don't have to provide a password. That seems cool. How secure is that? Using this option, the IndexedDB is still created with a password and the secret keys are still encrypted. However, the automatically created password is stored unencrypted within the localStorage of your browser. This is NOT considered secure and therefore should only be used when testing the app. I use a browser extension to manage my keys. Can I still use delegation.monster with this extension? Yes you can. delegation.monster supports browser extensions that implement NIP-07 . In that case you don't need to store any secret keys within delegation.monster. They remain within your browser extension.
